# 算数ふくしゅうクイズ — Webアプリ化 指示書

## これは何か
小学6年生の算数を復習するクイズアプリ。もともとClaude.ai上でHTML1枚のプロトタイプとして作り、動作確認済み。これをNext.jsのちゃんとしたWebアプリにして、Vercelにデプロイしたい。

同梱の `questionBank.js` に問題生成ロジック(6年生13カテゴリ、乱数で毎回違う問題を作る)が入っている。**このロジックはそのまま使ってOK。中身を書き換える必要はない。**

## やりたいこと(要件)

### 1. 新しいリポジトリとして作る
- Next.js (App Router) + Tailwind CSS で新規プロジェクトを作成
- Vercelにデプロイして、いつでも同じURLで開けるようにする
- GitHubの新しいプライベートリポジトリで管理する

### 2. 画面構成
- **トップ画面**:タイトル、「スタート」ボタン、「📊 にがてぶんやを見る」ボタン
- **クイズ画面**:1問ずつ表示。数値入力タイプと4択タイプの2種類がある(`questionBank.js` の `type` フィールドで判定: `'numeric'` か `'choice'`)。10問固定。正解/不正解のフィードバックと簡単な解説を出す
- **結果画面**:正解数、ランク(SS/S/A/B/C)、間違えた問題の一覧と解説、「もう一度」「にがてぶんやを見る」ボタン
- **にがてぶんや画面**:カテゴリごとの正答率を、正答率が低い順に一覧表示(棒グラフ的な見た目でOK)。データが無ければ「まだデータがありません」と表示

デザインは今のプロトタイプ(元HTML)を参考にしつつ、Tailwindで綺麗に作り直してOK。丸みのあるカード、パステル系の配色、大きめのタップ領域(スマホ操作前提)を意識してほしい。

### 3. スマホ対応必須
- メインの利用シーンはiPhoneのブラウザ
- タップ領域は大きめに、文字サイズは読みやすく
- 画面上部の余白は少し多めに(スマホのブラウザUIと問題文が重ならないように)

### 4. 成績データの保存 → Vercel KV を使う
- **どの端末から開いても同じ記録が見られるようにしたい**(ログイン機能は不要、個人利用の想定)
- Vercel KV(`@vercel/kv`)を使って、カテゴリ別の「挑戦回数」「正解数」を保存する
- データ構造の例:
  ```json
  {
    "対称な図形": { "attempts": 5, "correct": 4 },
    "円の面積": { "attempts": 3, "correct": 1 }
  }
  ```
- API Route(例: `/api/stats`)を作り、
  - `GET /api/stats` → 現在の成績データを返す
  - `POST /api/stats` → クイズ終了時に、その回の結果(カテゴリごとの正解/不正解)を送信し、サーバー側で加算保存する
- Vercel KVの作成方法:Vercelダッシュボードの該当プロジェクト → Storage → Create Database → KV を選択。環境変数は自動で設定されるはず

### 5. 同梱ファイル
- `questionBank.js`:問題生成ロジック(そのまま `lib/` 等に配置して import する想定)
  - `generateQuiz(count)` で指定した数の問題配列を取得できる
  - 各問題オブジェクトは `{ text, type, answer, choices?, explain, category, tolerance? }` の形
  - `type: 'numeric'` の場合、`tolerance` があれば許容誤差(小数の丸め対策)
  - `type: 'choice'` の場合、`choices` に4択の文字列配列が入っている

## 進め方の希望

- 迷ったら選択肢を提示してから進めてほしい
